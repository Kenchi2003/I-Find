1. Core Functions (จำเป็นมาก / ขาดไม่ได้)
1.1 Category Selection
•	เลือกหมวดหมู่สินค้า (Smartphone, Laptop, ฯลฯ)
•	ส่งค่า category id ไปหน้าอื่น
•	ใช้เป็น entry point หลักของผู้ใช้
สถานะ: ทำแล้ว
________________________________________
1.2 Keyword Search
•	ค้นหาสินค้าด้วย keyword
•	รองรับ use case เช่น
“โทรศัพท์เล่นเกม”, “โน้ตบุ๊กทำงาน”
สถานะ: UI มีแล้ว / Logic ยังไม่มี
________________________________________
1.3 Category Context Page
•	หน้าแสดงข้อมูลตามหมวดที่เลือก
•	อ่านค่า id จาก URL
•	แสดง title / description ตามหมวด
สถานะ: โครงสร้างมีแล้ว (Category.html)
________________________________________
1.4 Language Switch (TH | EN)
•	สลับภาษาได้ทั้งเว็บ
•	จำค่าภาษาเดิม
•	ใช้โครงสร้างเดียวกันทุกหน้า
สถานะ: กำลังทำ
________________________________________
2. Recommendation Logic (หัวใจของเว็บ)
2.1 User Preference Input
•	รับข้อมูลจากผู้ใช้ เช่น
o	งบประมาณ
o	จุดประสงค์การใช้งาน
o	สิ่งที่ให้ความสำคัญ (Performance / Camera / Battery)
________________________________________
2.2 Product Matching / Scoring
•	คำนวณคะแนนความเหมาะสม (% Match)
•	เปรียบเทียบหลายตัวเลือก
•	อธิบายเหตุผลของคะแนน
________________________________________
2.3 Result Display
•	แสดงรายการที่แนะนำ
•	เรียงจากเหมาะสมที่สุด
•	มีเหตุผลกำกับ (Explainable Recommendation)
________________________________________
3. Usability & UX Functions (ควรมี)
3.1 Responsive Design
•	ใช้งานได้ดีบนมือถือ / Tablet
•	Layout ไม่แตก
สถานะ: ทำแล้ว
________________________________________
3.2 Empty / Error State
•	ไม่มีผลลัพธ์
•	ค้นหาไม่เจอ
•	URL ผิด
________________________________________
3.3 Loading State
•	แสดง Loading เมื่อประมวลผล / เรียกข้อมูล
________________________________________
3.4 Navigation Feedback
•	Highlight category ที่เลือก
•	บอกผู้ใช้ว่า “คุณอยู่ตรงไหน”
________________________________________
4. Trust & Transparency (เว็บดูน่าเชื่อถือ)
4.1 Disclaimer
•	แจ้งว่าคำแนะนำเป็นข้อมูลประกอบการตัดสินใจ
________________________________________
4.2 Data Source Info
•	บอกว่าข้อมูลมาจากไหน
•	อัปเดตเมื่อใด
________________________________________
5. Analytics & Improvement (เพื่อพัฒนาในอนาคต)
5.1 Search Tracking
•	คำที่ถูกค้นหาบ่อย
•	หมวดที่คนสนใจ
________________________________________
5.2 Feedback System
•	ปุ่ม “คำแนะนำนี้ช่วยคุณได้ไหม?”
•	เก็บ feedback เพื่อปรับ AI / logic
________________________________________
6. Optional (เมื่อเว็บเริ่มโต)
6.1 User Profile
•	บันทึก preference
•	แนะนำได้แม่นขึ้น
________________________________________
6.2 Compare Products
•	เทียบสินค้า 2–3 ตัว
________________________________________
6.3 Affiliate / External Links
•	ลิงก์ไปซื้อ
•	Monetization
________________________________________
สรุปแบบผู้จัดการโปรเจกต์
ถ้าทำแค่ Phase 1 (MVP)
ให้โฟกัสเฉพาะ:
1.	Category → Input → Recommendation → Result
2.	Language (TH | EN)
3.	UX ชัด เข้าใจง่าย
สิ่งที่คุณทำอยู่ตอนนี้ มาถูกทางมากแล้ว และเหมาะกับการพัฒนาเป็นเว็บจริง
